# RFC 0002: Efferva Core Axioms

Status: Draft
Created: 2026-03-18

---

## Abstract

This RFC defines the core axioms of Efferva.
These axioms constrain all subsequent protocol specifications.

---

## Axioms

### Axiom 1: User Sovereignty Over Agents

Every person should own their own Agent.

Constraint:
- Agents must not require platform accounts to function
- Agent state must be exportable and portable
- Agent control must not depend on a single vendor

### Axiom 2: Intelligence Circulation by Use

Intelligence circulates by use, not by enclosure.

Constraint:
- Protocols must not create artificial scarcity of compute, models, or skills
- Access to intelligence resources must not require platform membership
- Intelligence capabilities must be composable across providers

### Axiom 3: Public Rules, Private Persons

Rules are public and auditable.
Persons remain private and untraceable.

Constraint:
- Protocol rules must be verifiable by any participant
- Personal identity must not be required for protocol participation
- Exchange verification must not expose participant identity

### Axiom 4: Memory Ownership

Memory belongs to individuals.

Constraint:
- Memory must not be uploaded by default
- Memory must be exportable in full
- Memory deletion must be irreversible and complete

### Axiom 5: Compatibility Without Dependence

New protocols interoperate with old systems.
Old systems are resource layers, not sovereignty layers.

Constraint:
- Protocols must be able to call existing APIs and services
- Protocols must not require old systems for core sovereignty functions
- Old system access must be optional, not mandatory

### Axiom 6: Replacement Over Patching

Efferva advances by replacing old protocol fulcrums, not by patching old systems.

Constraint:
- New protocols must rewrite necessary functions of old protocols
- New protocols must not depend on old protocols for core functions
- Compatibility layers must be clearly separated from sovereignty layers

---

## Non-Axioms

The following are not axioms:
- Specific encryption schemes
- Specific consensus mechanisms
- Specific identity systems
- Specific governance structures
- Specific economic models

These are implementation details.

---

## Rationale

These axioms prevent re-enclosure of intelligence resources.
